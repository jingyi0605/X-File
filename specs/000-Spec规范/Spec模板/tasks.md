# 任务清单 - <功能名称>（人话版）

状态：Draft

## 这份文档是干什么的

这份任务清单不是拿来堆术语的，是拿来让人快速知道：

- 先做什么
- 后做什么
- 每一步到底改什么
- 做完后怎么判断是不是真的完成了

## 状态说明

- `TODO`：还没开始
- `IN_PROGRESS`：正在做
- `BLOCKED`：被外部问题卡住
- `IN_REVIEW`：已经有结果，等复核
- `DONE`：已经完成，并且已经回写状态
- `CANCELLED`：取消，不做了，但要写原因

规则：

- 只有 `状态：DONE` 的任务才能勾选成 `[x]`
- `BLOCKED` 必须写清楚卡在哪里
- `CANCELLED` 必须写清楚为什么不做
- 每做完一个任务，必须立刻更新这里

## 写法要求

每个任务都必须优先回答下面这些问题：

1. 这一步到底做什么
2. 做完后你能看到什么
3. 依赖什么
4. 先看哪些文档
5. 主要改哪些文件
6. 这一步明确不做什么
7. 怎么验证

可以保留“对应需求”“对应设计”这些追踪信息，但它们只能是辅助，不能把正文写成一堆黑话。
