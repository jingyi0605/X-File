# Spec 空白模板

本目录是一套可复制的新 Spec 空白模板。

建议用法：

1. 复制整个 `specs/000-Sepc规范/Spec模板/` 目录
2. 重命名为新的 Spec 目录，例如 `specs/009-能力名称/`
3. 将以下占位符统一替换：
   - `<Spec编号>`
   - `<功能名称>`
   - `<需求名称>`
   - `<阶段名称>`
   - `<任务标题>`
   - `<设计章节编号>`
4. 按需补充 `docs/` 下的接口、联调、验收文档

创建新 Spec 时，建议同时参考：

- `specs/000-Sepc规范/Codex-Spec规范文档.md`
- `specs/000-Sepc规范/sepc-creation-lifecycle-management/`

本模板的目标：

- 快速建立标准目录结构
- 快速生成符合 Codex 使用习惯的三份主文档
- 保证任务具备独立执行所需的上下文信息

## 模板内容

- `requirements.md`：需求文档模板
- `design.md`：设计文档模板
- `tasks.md`：任务文档模板
- `docs/README.md`：补充文档目录说明

## 使用要求

复制后请至少确认以下事项：

1. 每条需求都有编号、用户故事、验收标准
2. `design.md` 的章节有稳定编号，可被任务引用
3. `tasks.md` 的每个任务都有状态、依赖、需求关联、设计关联、上下文入口、验收标准、验证方式
4. 任意打开一个任务，都能快速知道要看哪些需求、设计和补充文档
