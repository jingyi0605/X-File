# 20260608-当前CodingNS文档库迁移清单

这份清单说明从 CodingNS 迁到 X-File 时，哪些代码应该复刻，哪些代码不要搬。

原则很简单：

> 只搬文档库本身，不搬代码模式、事务模式外壳、Teable、Butler 和工作区会话。

## 1. 后端优先迁移

### 1.1 索引核心

这些是文档库真正的重活，应该迁到 X-File 自己的索引包里：

- `apps/host/src/modules/affairs-indexer/contracts/src/`
- `apps/host/src/modules/affairs-indexer/core/src/config/`
- `apps/host/src/modules/affairs-indexer/core/src/logging/`
- `apps/host/src/modules/affairs-indexer/core/src/parser/`
- `apps/host/src/modules/affairs-indexer/core/src/repositories/`
- `apps/host/src/modules/affairs-indexer/core/src/scanner/`
- `apps/host/src/modules/affairs-indexer/core/src/services/dirty/`
- `apps/host/src/modules/affairs-indexer/core/src/services/export/`
- `apps/host/src/modules/affairs-indexer/core/src/services/indexer/`
- `apps/host/src/modules/affairs-indexer/core/src/services/search/`
- `apps/host/src/modules/affairs-indexer/core/src/services/tagging/`
- `apps/host/src/modules/affairs-indexer/core/src/services/watch/`
- `apps/host/src/modules/affairs-indexer/core/src/sqlite/`
- `apps/host/src/modules/affairs-indexer/core/src/tagging/`
- `apps/host/src/modules/affairs-indexer/core/src/utils/`

迁移要求：

- 包名改成 X-File 自己的命名，例如 `@x-file/indexer`。
- 不能引用 CodingNS Host 私有路径。
- SQLite 继续使用 `better-sqlite3`，不要使用 `node:sqlite`。
- 原来的 `affairs` 命名可以在内部逐步改成 `library` / `document-library`。

### 1.2 文档库服务

这些是 X-File 后端 API 的主要来源：

- `apps/host/src/modules/workspace/affairs-library-service.ts`
- `apps/host/src/modules/workspace/affairs-library-controller.ts`
- `apps/host/src/modules/workspace/affairs-library-preview-link-service.ts`
- `apps/host/src/modules/workspace/affairs-library-dirty-watch-service.ts`
- `apps/host/src/modules/workspace/affairs-library-refresh-contract.ts`
- `apps/host/src/modules/workspace/affairs-library-debug-log.ts`
- `apps/host/src/modules/workspace/affairs-tag-service.ts`
- `apps/host/src/modules/workspace/affairs-tag-controller.ts`

迁移要求：

- `AffairsLibraryService` 改成 `LibraryService`。
- `workspaceId + userId` 模型改成 `libraryId + rootDir`。
- `AFFAIRS_GLOBAL_WORKSPACE_ID` 不要继续作为核心模型，可以只在兼容层里临时保留。
- 删除对 `WorkspaceService`、事务视图、工作台导航状态的依赖。
- 文件路径安全校验必须保留。
- 预览、下载、文件操作的行为要和当前一致。

### 1.3 后台任务和 watcher 规则

要复刻当前已经形成的规则：

- watcher 只打脏标记，不直接跑重活。
- 读接口只读缓存或快照，不顺手刷新。
- 同一个 `rootDir` 同类任务只允许一个 inflight。
- 目录 hint 是轻任务，不和全库索引抢同一个重锁。
- queue wait timeout、失败状态、最近完成时间要暴露给前端。
- 不同 `rootDir` 的重任务应该能隔离。

- `specs/spec016.1-事务视图文档库与索引HOST集成/`
- `specs/spec016.2-事务文档库外部变更感知与自动刷新/`
- `specs/spec016.4-事务文档库刷新阻塞治理与索引稳定性改造/`
- `specs/spec001.2-后端任务调度与主线程压力治理/20260412-后台任务接入规范.md`

## 2. 前端优先迁移

这些是当前独立文档库应用第一版最直接的前端来源：

- `apps/user-app/src/features/mobile-affairs-library/components/MobileLibraryBrowserView.tsx`

- `apps/user-app/src/features/mobile-affairs-library/components/MobileLibraryTagsView.tsx`

- 保留当前布局和样式，第一版不要重画。
- 所有展示文案迁到 X-File 的 i18n 字典。
- 不再依赖 CodingNS 的 mobile shell。

### 2.2 文档库状态 hook 和工具函数

这些逻辑要迁，但要改名和去耦合：

- `apps/user-app/src/features/workbench/hooks/useAffairsLibraryState.ts`

- 保留 `browseMode`、`viewMode`、`selectedFolderPath`、`selectedTagPaths`、`selectedDocumentId`、`librarySort`。
- 缓存 key 不再使用 `workspaceId`，改成 `libraryId`。

### 2.3 API 类型

当前文档库 DTO 在这些地方被使用：

- `apps/user-app/src/features/conversation/api/conversation-api`

迁移要求：

- 统一搬到 `packages/shared/src/library-types.ts`。
- 前后端都从 shared 包引用类型。
- API 路径从 `/api/affairs/library-*` 改成 `/api/library/*`。

## 4. 明确不要迁移

这些不要搬进 X-File 第一版：

- `apps/host/src/modules/workspace/workspace-service.ts`
- `apps/host/src/modules/workbench/`
- `apps/host/src/modules/sessions/`
- `apps/host/src/modules/terminal/`
- `apps/host/src/modules/git/`
- `apps/host/src/modules/plugins/`
- `apps/host/src/modules/butler/`
- `apps/host/src/modules/workspace/teable-*`
- `apps/user-app/src/features/workbench/components/AffairsWorkbenchView.tsx`
- `apps/user-app/src/features/mobile-affairs/`
- `apps/user-app/src/features/mobile-workspaces/`
- `apps/user-app/src/features/conversation/` 里和文档库 API 无关的会话逻辑

原因：

- X-File 是文档库应用，不是 CodingNS 的事务模式副本。
- 搬这些会把旧耦合结构带到新仓库。
- CodingNS 后续要做的是调用 X-File，不是让 X-File 长成另一个 CodingNS。

## 7. 迁移时最容易犯的错

1. **把 `workspaceId` 换个名字继续用。**
   - 新模型应该是 `libraryId/rootDir`，不是假装的工作区。
2. **把 watcher 做成扫描器。**
   - watcher 只报信，重活进后台任务。
3. **读接口顺手刷新。**
   - 这是性能问题的根。读就是读，刷新必须是显式入口或后台调度。
4. **把事务模式也搬过来。**
   - X-File 第一版只做文档库。
5. **为了快把前端文案硬编码。**
   - 新仓库也必须有统一 i18n。
