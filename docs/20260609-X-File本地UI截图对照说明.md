# 20260609-X-File本地UI截图对照说明

这份文档说明怎么在本机启动 X-File 的 server、web，绑定一个临时资料库 fixture，然后人工截 UI 对照图。

重点：这不是自动化 UI 测试。不要把 Playwright、截图产物或临时 fixture 加进业务依赖和 Git。

## 1. 这次要检查什么

截图至少覆盖 5 个状态：

1. 主界面 grid：三栏结构、目录树、中间网格、右侧详情。
2. list / Finder：列表密度、行高、文件夹/文件图标、选中态。
3. 右键菜单：文档、文件夹或空白处右键后的菜单结构。
4. 新建弹窗：新建目录或新建文件弹窗。
5. 标签弹窗：编辑文档标签或目录标签弹窗。

## 2. 准备临时资料库和截图目录

在仓库根目录执行：

```bash
node scripts/prepare-ui-screenshot-session.mjs
```

脚本只做两件事：

- 在系统临时目录创建一个文档库 fixture。
- 输出截图保存目录和绑定 API 示例。

它不会安装 Playwright，也不会修改 `package.json` 或 `pnpm-lock.yaml`。

请从输出里记住这两个路径：

- `libraryRoot`：后面绑定到 X-File 的资料库根目录。
- `screenshotsDir`：手工截图临时保存目录。

## 3. 启动 server

开一个终端，在仓库根目录执行：

```bash
pnpm dev:server
```

默认后端地址：

```text
http://127.0.0.1:17321
```

另开一个终端检查健康状态：

```bash
curl -sS http://127.0.0.1:17321/api/health
```

看到 `ok: true` 就继续。不要关闭这个 server 终端。

## 4. 启动 web

再开一个终端，在仓库根目录执行：

```bash
pnpm dev:web
```

默认前端地址：

```text
http://127.0.0.1:17320
```

用浏览器打开这个地址。

## 5. 绑定临时资料库

可以走页面，也可以走 API。为了复现更稳定，建议先用 API 绑定。

把下面命令里的 `/path/to/libraryRoot` 换成第 2 步脚本输出的 `libraryRoot`：

```bash
curl -sS -X PUT http://127.0.0.1:17321/api/library/binding \
  -H 'content-type: application/json' \
  --data '{"rootDir":"/path/to/libraryRoot"}'
```

然后触发一次刷新：

```bash
curl -sS -X POST http://127.0.0.1:17321/api/library/refresh \
  -H 'content-type: application/json' \
  --data '{"reason":"manual_ui_screenshot"}'
```

刷新后等几秒，再检查快照：

```bash
curl -sS http://127.0.0.1:17321/api/library/snapshot
```

确认能看到 `binding` 和文档数量后，刷新浏览器页面。

## 6. 截图清单

截图全部保存到第 2 步输出的 `screenshotsDir`。文件名建议固定，方便别人对照。

### 6.1 主界面 grid

1. 进入文档库主界面。
2. 切到 grid / 网格视图。
3. 选中一个 Markdown 文件，让右侧详情出现内容。
4. 截图保存为：

```text
01-grid-main.png
```

重点看：左侧栏宽度、中间网格密度、右侧详情面板、选中态。

### 6.2 list / Finder

1. 切到 list / Finder 视图。
2. 进入 `产品资料/需求` 或 `团队资料/会议纪要`。
3. 选中一行文档。
4. 截图保存为：

```text
02-list-finder.png
```

重点看：行高、图标、文件名和摘要信息是否像 CodingNS 文档库。

### 6.3 右键菜单

1. 在文档项上右键。
2. 如果要覆盖空白区域菜单，进入 `空目录-用于右键空白处` 后在空白处右键。
3. 保持菜单展开状态截图。
4. 截图保存为：

```text
03-context-menu.png
```

重点看：菜单位置、阴影、分割线、`新建` 子菜单、`标签` 入口和危险操作样式。

### 6.4 新建弹窗

1. 通过右键菜单或页面操作打开“新建目录”或“新建文件”。
2. 不要真正创建也可以，只要弹窗完整展示。
3. 截图保存为：

```text
04-create-modal.png
```

重点看：弹窗标题、字段说明、输入框、取消按钮和创建按钮是否接近统一模态框规范。

### 6.5 标签弹窗

1. 选中文档 `标签弹窗检查.md`。
2. 打开“编辑标签”或右键菜单里的“标签”。
3. 保持弹窗打开，必要时输入 `截图检查,文档库`。
4. 截图保存为：

```text
05-tag-modal.png
```

重点看：标签输入说明、推荐标签、保存按钮、弹窗间距。

## 7. 当前还没有达到像素级的点

这轮人工对照用于确认结构和主要交互，不承诺像素级一致。仍要继续压的点：

- 中间网格列距和卡片密度还要继续对 CodingNS 当前截图。
- Finder/list 行高、摘要文本截断和选中态细节还要继续微调。
- 右侧详情空态的尺寸、上下间距和图标重量还没到像素级。
- 右键菜单子菜单展开位置在不同浏览器缩放下仍要复核。
- macOS Tauri 实机玻璃效果不在本 web 截图里完成最终判断，需要另走桌面窗口截图。

## 8. 清理

截图结束后必须停掉开发服务器：

1. 回到 `pnpm dev:web` 终端，按 `Ctrl+C`。
2. 回到 `pnpm dev:server` 终端，按 `Ctrl+C`。
3. 删除第 2 步脚本输出的 `sessionRoot` 临时目录。

示例：

```bash
rm -rf /tmp/x-file-ui-xxxxxx
```

不要提交：

- `screenshotsDir` 里的截图。
- `libraryRoot` 里的 fixture 文件。
- `.x-file-dev/` 这类本地运行状态文件。
