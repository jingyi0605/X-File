# 20260608-X-File第一版验收说明

这份文档验收第一版代码完成状态：桌面壳、系统托盘菜单、后端子进程托管入口、HTTP 服务状态、常驻策略、CodingNS 集成入口、打包校验脚本、Tauri updater/签名配置占位和 Windows runner 工作流。它不声称安装包已发布，也不声称 Windows 已完成真机验证。

## 1. 基础构建

```bash
pnpm install
pnpm build
```

如果只验收桌面和服务端骨架：

```bash
pnpm --filter @x-file/server build
pnpm --filter @x-file/desktop build
```

`@x-file/desktop build` 会执行 TypeScript 编译和 `cargo check --manifest-path src-tauri/Cargo.toml`。本机需要安装 Rust 和 Tauri 2 需要的系统依赖。

## 2. 启动后端

不要为了验收主动启动长期 dev server。需要人工验证 API 时，再运行：

```bash
pnpm dev:server
```

默认地址：

```text
http://127.0.0.1:17321
```

## 3. HTTP 服务状态 API

读取状态：

```bash
curl http://127.0.0.1:17321/api/server/state
```

保存配置：

```bash
curl -X PUT http://127.0.0.1:17321/api/server/state \
  -H 'Content-Type: application/json' \
  -d '{"enabled":true,"host":"127.0.0.1","port":17321,"persistent":true}'
```

验收点：

- 默认 host 是 `127.0.0.1`。
- 返回字段包含 `enabled`、`host`、`port`、`persistent`、`running`、`lifecycleState`、`startedAt`、`lastError`、`persistentPolicy`。
- 传入非 `127.0.0.1` 的 host 会返回错误。
- `PUT /api/server/state` 在正式 server 生命周期模式下会按配置启动或停止本机 HTTP 监听；测试/嵌入模式可以只保存配置。

## 4. 常驻策略 API

```bash
curl http://127.0.0.1:17321/api/server/state
```

验收点：

- `persistent=true` 且 `enabled=true` 时，`persistentPolicy.keepBackendOnWindowClose=true`。
- 否则关闭窗口按普通退出处理。
- 用户明确退出应用时，策略始终要求停止 HTTP 服务。
- 第一版桌面壳已经实现关闭窗口隐藏主窗口，`persistentPolicy.implementedByDesktopShell=true` 可以如实返回；托盘菜单可以恢复窗口或退出应用。

## 5. CodingNS 集成入口

```bash
curl http://127.0.0.1:17321/api/integration/status
```

验收点：

- 返回 `ok=true`。
- 返回 HTTP 服务状态。
- 返回文档库快照摘要，不要求 CodingNS 读取 X-File 内部文件。
- X-File 未运行时，CodingNS 应提示用户启动 X-File，不应退回本地重扫描。

## 6. 桌面壳骨架

关键文件：

- `apps/desktop/src-tauri/tauri.conf.json`
- `apps/desktop/src-tauri/Cargo.toml`
- `apps/desktop/src-tauri/src/main.rs`

验收点：

- 配置是 Tauri 2。
- 包含 Tauri updater 配置占位：`plugins.updater.pubkey`、`plugins.updater.endpoints` 和 `bundle.createUpdaterArtifacts=true`。
- `plugins.updater.pubkey` 当前仍是占位值，真实发布前必须替换为 Tauri CLI 生成的公钥。
- Rust 侧已经初始化 `tauri-plugin-updater`，但没有真实更新清单、签名私钥和发布服务器前，不能说自动更新已经完成实机验收。
- 签名 secrets 不写入仓库，只能从环境变量或 GitHub Secrets 注入。
- Tauri 命令返回结构化的常驻策略、HTTP 服务说明和后端托管状态，并实现关闭窗口隐藏主窗口。
- 系统托盘菜单包含显示窗口、隐藏窗口、开启/关闭后端常驻、启动/停止内置后端和退出 X-File。
- 后端托管入口默认执行 `node x-file-server/main.js` 一类资源入口；发布包仍需要携带 Node runtime，或后续改成真正的 Tauri sidecar。
- `pnpm --filter @x-file/desktop build` 必须能完成 TypeScript 编译和 `cargo check`。

## 7. 签名和自动更新前置检查

普通结构检查：

```bash
node scripts/check-desktop-release-secrets.mjs
```

发布前强制检查：

```bash
node scripts/check-desktop-release-secrets.mjs --platform windows --require-real-secrets
```

真实发布需要这些配置：

- `TAURI_SIGNING_PRIVATE_KEY`：Tauri updater 私钥。
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`：Tauri updater 私钥密码；如果私钥无密码，也要显式配置为空字符串。
- `X_FILE_UPDATER_PUBLIC_KEY`：Tauri updater 公钥，必须和 `tauri.conf.json` 的 `plugins.updater.pubkey` 一致。
- `X_FILE_UPDATER_ENDPOINT`：真实更新清单地址，必须和 `tauri.conf.json` 的 updater endpoint 一致。
- `WINDOWS_SIGNING_CERTIFICATE`：可选。Windows 代码签名证书，建议使用 base64 PFX；不影响 Tauri updater 验签更新，只影响安装包发布者可信度和 SmartScreen 体验。
- `WINDOWS_SIGNING_CERTIFICATE_PASSWORD`：可选。Windows 代码签名证书密码。

Updater 私钥、公钥和更新地址没有真实配置前，只能说明“配置占位和检查脚本存在”，不能说已经完成自动更新。Windows 代码签名证书不是 updater 正常更新的必要条件。

## 8. 打包校验脚本

macOS 骨架校验：

```bash
pnpm build:macos
```

Windows 骨架校验：

```bash
pnpm build:windows
```

这两个脚本只校验 `@x-file/shared` 构建和 `@x-file/desktop` build/cargo check，不替代完整 server/web 构建或测试文件编译修复。设置 `X_FILE_REQUIRE_RELEASE_SECRETS=1` 后，脚本会强制检查签名和 updater secrets。Windows MSI 正式产物应在 Windows runner 上执行 `pnpm --filter @x-file/desktop tauri:build`。

## 9. Windows runner 工作流

工作流文件：

```text
.github/workflows/x-file-windows-build.yml
```

验收点：

- 只能手动触发，避免没有 runner 或 secrets 时误跑。
- 默认使用 `self-hosted`、`Windows` 和 `x-file-windows` 标签。
- 会先执行 `node scripts/check-desktop-release-secrets.mjs --platform windows --require-real-secrets`。
- 会执行 `pnpm --filter @x-file/web build` 和 `pnpm --filter @x-file/desktop tauri:build`。
- 会上传 MSI、NSIS 和 updater 签名产物；没有产物时失败。

没有真实 self-hosted Windows runner 和真实 secrets 前，不能把这个工作流当成已经完成 Windows 实机构建。

## 10. 第一版代码完成边界

已经完成：

- 常驻策略有可读状态，桌面壳会在常驻开启时隐藏关闭的主窗口；系统托盘菜单和后端子进程托管入口已完成代码接入。
- Tauri updater 和签名配置占位已补，发布前置检查脚本已补。
- Windows runner GitHub Actions 工作流已补，但还需要真实 runner 和 secrets 才能验收。
