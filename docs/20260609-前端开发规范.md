# X-File 前端开发规范

本文档用于约束 X-File 前端页面、组件、弹窗、表单与按钮的开发方式。进行任何前端代码编写前，必须先阅读本文档，并优先复用现有全局样式体系。

## 1. 总原则

- X-File 前端必须复刻并延续 CodingNS 主仓库的全局样式体系。
- 新增 UI 不允许先写局部魔法值；必须先检查 `apps/web/src/styles.css` 是否已有可复用 token、统一类名或组件样式。
- 如果现有全局样式不能覆盖需求，先扩展全局 token 或统一类名，再让页面和组件消费它们。
- 保持组件代码简单，视觉规则沉到全局样式，避免每个页面自己修一遍按钮、表单、弹窗尺寸。

## 2. 开发前检查清单

写前端代码前必须检查：

1. 当前需求是否是页面、表单、按钮、弹窗、标签、空状态、提示条、列表、卡片或双栏布局。
2. `apps/web/src/styles.css` 是否已有对应全局 token 或统一类名。
3. 是否能复用 `DesktopModal`、`ModalAtoms`、`.button-row`、`.primary-button`、`.secondary-button`、`.danger-button`。
4. 是否存在 CSS Grid 或 Flex 默认拉伸导致控件尺寸异常的风险。
5. 是否会破坏现有页面的按钮高度、表单高度、弹窗材质和主仓库视觉一致性。

没有完成这些检查，不要开始写局部样式。

## 3. 按钮规范

按钮统一使用以下类名：

- `.primary-button`：主操作按钮，必须是蓝底白字。
- `.secondary-button`：次操作按钮，使用统一浅色背景与普通文字颜色。
- `.danger-button`：危险操作按钮，使用统一危险色 token。

按钮高度统一由全局 token 管理：

- `--app-button-height`
- `--app-button-padding-x`
- `--app-button-radius`
- `--app-button-primary-*`
- `--app-button-danger-*`

禁止在单个页面或组件中随手写按钮高度，例如：

```css
/* 禁止：局部魔法值会破坏统一体系 */
.some-page button {
  height: 58px;
  padding: 20px;
}
```

按钮行必须优先使用 `.button-row`。`.button-row` 的职责是统一按钮排列、间距、垂直对齐和防止按钮被父布局拉伸。

## 4. 表单和控件规范

表单控件统一消费：

- `--control-height-sm`
- `--control-height-md`
- `--control-height-lg`
- `--modal-control-*`

输入框、选择框、文本域应优先使用全局 `input`、`select`、`textarea` 样式。若需要新增控件能力，先补全全局 token，再在组件中复用。

## 5. 布局规范：避免 Grid/Flex 拉伸控件

CSS Grid 默认 `align-items: stretch`，这会把同一行里的子元素拉到同样高度。双栏布局中，如果右侧 logo、插图、预览卡片较高，左侧表单也会被拉高；表单内部的按钮行可能继续被撑成异常大按钮。

因此，双栏表单、初始化页、设置页、带插图的卡片布局必须显式处理对齐：

```css
.app-split-layout {
  display: grid;
  align-items: start;
}

.app-form-stack {
  display: grid;
  align-content: start;
}
```

如果页面使用现有局部类，也必须保证同等语义：

```css
.some-page-body {
  display: grid;
  align-items: start;
}

.some-page-form {
  display: grid;
  align-content: start;
}
```

这不是可选项。凡是左表单、右插图或左右卡片布局，都必须防止默认 stretch 把控件撑坏。

## 6. 模态框规范

弹窗必须优先使用：

- `DesktopModal`
- `ModalAtoms`
- `.modal-actions`
- `.modal-field`
- `.modal-section`
- `.modal-empty-state`
- `.modal-tag`

模态框视觉统一消费：

- `--modal-backdrop-*`
- `--modal-frame-*`
- `--modal-panel-*`
- `--modal-control-*`
- `--modal-button-*`

禁止为单个弹窗随意重写背景、边框、阴影、控件高度和按钮高度。如果确实需要新能力，先扩展 `apps/web/src/styles.css` 中的全局 token。

## 7. 新增样式的正确流程

新增前端样式时按以下顺序：

1. 先查 `apps/web/src/styles.css` 是否已有可复用样式。
2. 能复用就复用，不新增局部规则。
3. 不能复用时，先判断是否应扩展全局 token 或全局类名。
4. 页面局部 CSS 只写布局语义，不写破坏体系的颜色、尺寸和阴影魔法值。
5. 完成后至少运行：

```bash
pnpm --filter @x-file/web build
```

如只改 TypeScript 逻辑，也应运行：

```bash
pnpm --filter @x-file/web typecheck
```

## 8. 当前已知教训

初始化页曾出现底部三个按钮高度异常的问题。根因是 `.library-init-body` 使用 CSS Grid 后默认 `align-items: stretch`，左侧表单被右侧 logo 面板高度拉伸，`.button-row` 里的按钮也被撑成大块。

此类问题以后不能靠截图后再单独修。所有类似页面必须在布局层面提前处理：

- 双栏容器使用 `align-items: start`。
- 表单容器使用 `align-content: start`。
- 按钮行使用 `.button-row`。
- 按钮高度只来自 `--app-button-height`。
- 主操作按钮必须保持 `.primary-button` 的蓝底白字语义。
